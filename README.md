# pari-mixer-scraper

Веб-приложение, которое собирает данные по матчам турнира Dota 2 (по
умолчанию — PARI Mixer Cup, league_id `19924`,
https://www.dotabuff.com/esports/leagues/19924-pari-mixer-cup) в локальную
базу SQLite и показывает их в браузере: выбираешь команду — видишь ники
игроков и героев, на которых они играли.

## Источники данных

Прямой скрейпинг HTML dotabuff.com не используется — сайт закрыт Cloudflare
(JS-челлендж "Just a moment...") для любых автоматических запросов. Вместо
этого используются три источника, и роли у них разные:

1. **Steam Web API** (`IDOTA2Match_570/GetMatchHistory`) — официальный API
   Valve, тот же, что использует сама Dota 2 / Dota Plus. Самый дешёвый
   способ получить список матчей лиги: один запрос отдаёт до 100 матчей
   сразу вместе с составами и героями. Требует бесплатный ключ, см. ниже.
   Результат матча (кто победил) он **не отдаёт**.
2. **mixer-cup.gg (GraphQL)** — источник правды по всему, чего нет в Dota:
   названия команд, составы, MMR, **результаты** (WIN1/WIN2), замены,
   очередь. Дёшево, несколько запросов за цикл. Он же перечисляет `matchId`
   всех игр турнира — это запасной путь поиска матчей, если Steam недоступен.
3. **[OpenDota API](https://docs.opendota.com/)** — только детали по
   конкретному `match_id`: драфт (picks/bans), стороны, `radiant_win`.
   По одному запросу на матч, поэтому это самое узкое место.

**Если ключ Steam недоступен**, список матчей приходится собирать из
mixer-cup и добирать каждый матч из OpenDota — по запросу на матч. На
шаред-IP (например, на free-тарифе Render) это упирается в rate-limit.
Признак в логах: `MixerCup match seed` тянет сотни матчей вместо нуля.

> Диагностика ключа: Steam отвечает голым `403` на **любой** эндпоинт, если
> ключ отозван/протух — это выглядит как «Valve закрыл API», но это не так.
> Проверьте тем же ключом `ISteamUser/GetPlayerSummaries`: если и там `403` —
> дело в ключе. Запрос **без** ключа при этом даёт `400`, а несуществующий
> метод — `404`; если так, значит до Steam вы доходите.

Намеренно **не** используются (проверено на этой лиге):
`/leagues/{id}/matches` — пусто, лига tier `excluded`; `/teams/{id}` —
отдаёт `200` с пустым телом; `/proPlayers` — ~4.4 МБ ради ~2 совпадений
из 10 (микс — это не про-игроки, а имена и так есть в ростерах mixer-cup).

### Как получить Steam Web API ключ

1. Откройте https://steamcommunity.com/dev/apikey (нужен Steam-аккаунт).
2. Зарегистрируйте ключ (domain можно указать `localhost`).
3. Скопируйте `.env.example` в `.env` и впишите ключ туда — **не в чат и не
   в код**:

   ```
   cp .env.example .env
   # затем откройте .env и впишите STEAM_API_KEY=...
   ```

## Установка

```bash
cd pari-mixer-scraper
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements.txt
```

## Запуск веб-приложения

**Windows, без терминала:** дважды кликните [`start.bat`](start.bat) в папке
проекта — он запустит сервер и через пару секунд сам откроет
http://127.0.0.1:5000 в браузере. Чтобы остановить — закройте открывшееся
чёрное окно консоли или нажмите в нём Ctrl+C.

**Через терминал** (PowerShell/cmd/bash — если нужно видеть логи или
приложение уже не через `.venv\Scripts\activate`):

```powershell
cd pari-mixer-scraper
.venv\Scripts\python.exe app.py
```

Откройте http://127.0.0.1:5000 в браузере. Сбор данных запускается сам в
фоне (см. «Автообновление матчей» ниже) — прогресс виден в шапке страницы,
слева появляется список команд, по клику показываются игроки и герои.

Сбор идемпотентен: повторные запуски докачивают только новые матчи, старые
не перезапрашиваются. Останов сервера — Ctrl+C в терминале.

## Деплой в облако (чтобы другие люди могли зайти по ссылке)

Локальный запуск виден только на вашем компьютере. Чтобы дать доступ другим —
разверните на [Render](https://render.com) (бесплатный тариф, поддерживает
Python из коробки). Шаги с созданием аккаунтов должны быть сделаны вами лично
— агент не может создавать аккаунты или логиниться от вашего имени.

1. **Создайте репозиторий на GitHub** (если ещё нет аккаунта — зарегистрируйтесь
   на github.com) и запушьте туда код:
   ```bash
   cd pari-mixer-scraper
   git remote add origin https://github.com/<ваш-username>/<repo-name>.git
   git branch -M main
   git push -u origin main
   ```
2. **Зарегистрируйтесь на [render.com](https://render.com)** (можно через GitHub).
3. **New → Blueprint** → выберите ваш репозиторий. Render сам прочитает
   [`render.yaml`](render.yaml) из корня проекта и настроит сборку/запуск.
4. Render попросит ввести значение переменной `STEAM_API_KEY` — впишите свой
   ключ в его дашборде (не в код, не в репозиторий).
5. Дождитесь деплоя — Render даст постоянную ссылку вида
   `https://pari-mixer-scraper.onrender.com`, которую можно скидывать кому угодно.

**Важные особенности бесплатного тарифа Render:**
- Сервис "засыпает" после ~15 минут без запросов и просыпается по первому
  обращению (~30-60 сек на холодный старт) — это нормально для бесплатного
  тарифа, а не баг.
- Диск на бесплатном тарифе не постоянный: `tournament.db` может обнуляться
  при перезапуске/пробуждении. Приложение это обрабатывает само — при
  старте, если база пустая, автоматически запускается сбор данных в фоне
  (см. `_auto_collect_if_empty` в [app.py](app.py)), так что сайт сам
  восстанавливает данные за 1-3 минуты без ручных действий.
  (Пробовали переехать на хостед-БД Turso/libSQL для постоянного хранения —
  откатили: их Rust-драйвер блокирует GIL при сетевых запросах настолько,
  что всё приложение переставало отвечать на время сборки данных.)

## Автообновление матчей

Данные обновляются сами, без ручного управления, через два механизма:

1. **Фоновый таймер внутри приложения** — раз в `COLLECT_INTERVAL_SECONDS`
   (по умолчанию 600 сек = 10 мин) сам запускает сбор. Работает только пока
   процесс жив.
2. **[GitHub Actions](.github/workflows/keep-alive.yml)** — раз в ~10 минут
   дёргает `POST /api/collect` на задеплоенном сайте (адрес берётся из
   переменной репозитория `APP_URL`: Settings → Secrets and variables →
   Actions → Variables). Это не только триггерит сбор, но и не даёт
   бесплатному Render уйти в сон между визитами, так что таймер из пункта 1
   продолжает тикать.

Изменить интервал внутреннего таймера — задать переменную окружения
`COLLECT_INTERVAL_SECONDS` в Render (0 — выключить). Изменить расписание
GitHub Actions — поправить `cron` в `.github/workflows/keep-alive.yml`.

## CLI (без браузера)

Тот же сбор данных и текстовая сводка доступны без веб-интерфейса:

```bash
python -m pari_mixer_scraper.collect --league-id 19924 --db tournament.db
python -m pari_mixer_scraper.report --db tournament.db
```

## Схема базы данных

- `heroes(hero_id, name, localized_name)`
- `teams(team_id, name)`
- `players(account_id, name, team_id)`
- `matches(match_id, league_id, start_time, duration, radiant_team_id, dire_team_id, radiant_win)`
- `match_players(match_id, account_id, hero_id, team_id, is_radiant, kills, deaths, assists)`

`match_players` — основная таблица для вопроса "какой игрок на каком герое
играл"; на ней строятся и веб-эндпоинты, и `report.py`.

## API веб-приложения

- `GET /api/teams` — список команд с числом игроков.
- `GET /api/teams/<team_id>` — игроки команды и герои, на которых каждый играл.
- `POST /api/collect` — запускает сбор данных в фоновом потоке (409, если уже запущен).
- `GET /api/collect/status` — статус текущего/последнего сбора и лог.

## Если league_id не подходит

- Проверить название лиги: `GET https://api.opendota.com/api/leagues/{league_id}`.
- Список всех лиг для поиска по имени: `GET https://api.opendota.com/api/leagues`.
- `leagueid` конкретного матча: `GET https://api.opendota.com/api/matches/{match_id}`.

## Ограничения

- Игроки, скрывшие профиль (анонимный `account_id`), не сопоставляются между
  матчами — такие записи пропускаются с предупреждением в логе.
- OpenDota, бесплатный тариф: 60 запросов/мин, 50k/месяц — клиент сам
  выдерживает паузы и повторяет запрос при 429.
- Steam Web API также ограничен по частоте запросов; клиент выдерживает
  паузу между запросами и делает retry при 429.
